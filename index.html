Why Angelica

Built differently.

Free. Permanently.

MIT license. Use Angelica in commercial projects, academic research, or internal tooling without fees, expiry dates, or usage limits. The code is yours — read it, modify it, ship it.

Accuracy you can verify.

Every result is benchmarked against published EPANET reference solutions. The solver's performance is documented, reproducible, and open to inspection — not a black box.

An architecture with a plan.

The current incompressible solver is the first stage of a platform designed to grow. Non‑isothermal, compressible, and multiphase pipeline simulation are all on the roadmap — not afterthoughts.

The workflow engineers already know.

Define sources and sinks, connect them through junctions, assign components, and run. No scripting, no configuration files, no setup rituals. From network to results in minutes.

Capabilities

A complete toolset for pipe network engineering.

Pipes

Darcy-Weisbach with Colebrook-White or Hazen-Williams closure. Arbitrary length, diameter, and wall roughness.

Fittings

Local-loss accessories from a built-in library or a custom K. Elbows, valves, tees, and any fitting with a known loss coefficient.

Pumps

Single-point EPANET model and piecewise-linear multi-point curves. Operating point computed directly from the required head.

Boundary conditions

Pressure and mass-flow boundaries on any node. Mixed types — sources, sinks, flow inlets, flow outlets — on the same network.

Graphical interface

Build and edit networks without writing code. Inspect convergence, review nodal and link results, and export a spreadsheet report.

Open source

Full source on GitHub under the MIT license. No subscriptions, no license fees, no vendor lock-in. Use it, fork it, extend it.

Validation

Accuracy you can check.

Hanoi benchmark

32-junction, 34-pipe network solved with Darcy-Weisbach and Colebrook-White. Nodal heads and link flows validated against the published EPANET reference solution.

EPANET pump benchmark

Pump-bearing network adapted from the official EPANET tutorial. Pump operating point, nodal pressures, and link flows validated against the published reference.

Tutorial suite

Eight cases covering pipes, fittings, elevation, inlet-flow, outlet-flow, and combined boundaries — for smoke testing, onboarding, and regression checks.

The pressure-correction algorithm, element laws, and solver strategy are fully documented on the methods page.
