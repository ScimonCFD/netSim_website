Capabilities

Everything needed to simulate a pipe network.

Pipes

Darcy-Weisbach friction with Colebrook-White or Hazen-Williams closure. Arbitrary length, diameter, and wall roughness.

Fittings

Local-loss accessories from a built-in library or a custom loss coefficient. Elbows, valves, tees, and any fitting with a known K.

Pumps

Single-point EPANET curve model and piecewise-linear multi-point curves. Flow computed directly from the required operating head.

Boundary conditions

Pressure and mass-flow boundaries on any node. Mixed types — pressure sources, pressure sinks, flow inlets, flow outlets — on the same network.

Graphical interface

Build and edit networks without writing code. Inspect convergence, review nodal and link results, and export a spreadsheet report.

Open source

MIT license. Full source on GitHub. No subscriptions, no license fees, no vendor lock-in. Use it, fork it, extend it.

Workflow

GUI-first, from network to results.

01

Launch the GUI

Open the graphical interface and start a new network or load a validated tutorial case.

02

Build the network

Add sources, sinks, and junctions, then connect them with links and assign pipes, fittings, or pumps.

03

Configure the case

Define the material, pressure-drop model, and numerical settings from the GUI.

04

Run and export

Run the solver, inspect convergence, review nodal pressures and link flows, and export the results to a spreadsheet.

Validation

Benchmarked against published reference solutions.

Hanoi benchmark

32-junction, 34-pipe network solved with Darcy-Weisbach and the Colebrook-White equation. Nodal heads and link flows validated against the published EPANET reference solution.

EPANET pump benchmark

Pump-bearing network adapted from the official EPANET tutorial. Pump operating point, nodal pressures, and link flows validated against the published reference.

Tutorial suite

Eight cases covering pipes, fittings, elevation changes, inlet-flow, outlet-flow, and combined boundaries — available for smoke testing and onboarding.

The pressure-correction algorithm, element laws, and solver strategy are documented in detail on the methods page.
